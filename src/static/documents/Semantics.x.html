Semantics of Propositional Logic

This section will discuss the semantics of propositional logic.

Meaning to $\mathcal{L}_1$-expressions is provided by $\mathcal{L}_1$-structures: they assign truth values to sentence letters.

Although $\mathcal{L}_1$-structures are infinite lists of truth-values, we can process their relevant parts in LEAN. As an example, we consider here an $\mathcal{L}_1$ structure which assigns falsity to sentence letters $P$, $Q$, and truth to $R$. You can see this in the code editor. By applying the definition of satisfaction, you can now evaluate the truth value of compound sentences. Just like as you would do with truth tables, LEAN computes the value of complex sentences.

You have to be precise with your brackets in LEAN. If you use multiple connectives and don't specify the brackets, LEAN automatically adds brackets from left to right. The first example shows this: if you input $P \land \neg P \lor R$, LEAN will evaluate $(P \land \neg P) \lor R$.

Countermodels

We just looked at a toy model in the previous example, with given truth values for $P,Q,R$. Next, let's consider countermodels. As a reminder, a sentence $\phi$ is logically implied by a set of sentences $\Gamma$ (formally $\Gamma \vDash \phi$) if there is no counterexample to the argument.
